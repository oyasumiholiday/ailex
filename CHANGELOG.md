# Changelog

すべての変更は dogfood（実際に書いて痛点を出す）か AI 実測（EXPERIMENTS.md）に根拠を持つ。

## 0.5.3 — 2026-07-13

- `run` の契約検査を**両バックエンド**で実施（従来はインタプリタのみ。JS 変換側の不一致を `backend_mismatch` として報告）
- ユーザの `eg` 契約が、そのプログラム自身による変換器の検証を兼ねる構造に
- 根拠: dogfood 第5R の parseInt 事故（インタプリタ合格・JS だけ失敗）と同型の穴が `run` に残っていた

## 0.5.2 — 2026-07-13

- `map` が `Option[T]` にも効く（`map(o, f) -> Option[U]`）
- 根拠: AI実測 A3 で両モデル（Haiku/Opus）の唯一の失敗が「map を Option に使った」同一原因。モデルの事前分布に言語を合わせる修正（A1→A2 と同型）。適合テスト 85→89

## 0.5.1 — 2026-07-12

- fold への期待型伝播: `fold(v, [], ...)` / `fold(v, none, ...)` が自然に書ける
- `[]` が check 位置（引数・契約）で型が付く
- String の `+`（文字列連結。concat のネスト解消）
- 修正: JS バックエンドの `parseInt` が常に none（テンプレートリテラルの `\d` 化け）— 両バックエンド不一致を conformance の JS 経路 evals で固定
- 根拠: dogfood 第5ラウンド（成績表 `examples/gradebook.ax`）。適合テスト 78→85

## 0.5.0 — 2026-07-12

- `Option[T]`: `some(x)` / `none` / `isSome` / `unwrapOr(o, default)`
- 失敗しうる stdlib は Option を返す: `find(v, 述語)` / `parseInt(s)` / `parseFloat(s)`
- `none` は文脈から型が決まる位置でのみ書ける（双方向型検査・診断で誘導）
- Option 値の契約（`eg f([]) = none` 等）も深い等価で判定
- 根拠: probe M（文字列→数値が一切書けない）・N（find に番兵レコードが要る）。適合テスト 67→78。受け入れ（aigen）8/8 退行なし

## 0.4.0 — 2026-07-11

- 文字列 stdlib: `split` / `join` / `contains` / `substring` / `trim` / `toString`（多相）
- 安全な既定値つき取得: `headOr(v, d)` / `getOr(v, i, d)`
- CLI: 実行時エラーで構造化診断 `{code: "runtime"}` を返す（以前は Node のスタックトレースでクラッシュ）
- リリース整備: `PRIMER.md`（AI 向け正典）、npm bin（`bin/ailex.js`）、パッケージ名 `ailex` v0.4.0
- 根拠: dogfood 第3ラウンド（probe 4本）。適合テスト 59→67

## 0.3.2 — 2026-07-11

- **ラムダの型注釈を任意化**（期待型から推論・注釈があれば照合）
- 根拠: AI 実測 A1 で Haiku/Opus の pass@1 失敗が全て「注釈なしラムダの parse エラー」→ 修正後 A2 で両モデル pass@1 100%・修復ゼロ
- 適合テスト 52→59

## 0.3.1 — 2026-07-11

- **意味論修正**: `==` / `!=` を構造等価に（リスト・レコードを深く比較。従来は参照比較で `[1.0] == [1.0]` が false）
- 型エイリアス `type Name = Ty`（透明・宣言前置）
- 根拠: dogfood 第2ラウンド（家計簿）。適合テスト 46→52

## 0.3.0 — 2026-07-10

- 構造的レコード: 型 `{x : Float}`・リテラル `{x = 1.0}`・アクセス `p.x`（連鎖可）
- 未知フィールド診断は使えるフィールド一覧を開示
- 契約 `eg` の判定を深い等価に修正（リストを返す eg の誤判定を解消）
- 適合テスト 36→46

## 0.2.0 — 2026-07-10

- 高階関数の正式サポート: 無名関数 `fn (x : T) => e`・第一級関数値（両バックエンド・健全）
- `map` / `filter` / `fold`
- 適合テスト 33→36

## 0.1.1 — 2026-07-10

- **健全性修正**: 高階関数が「型は通るがインタプリタで落ちる」バックエンド不一致だったため、型検査で関数値を一時禁止（0.2.0 で正式サポートに置換）
- 適合テスト 30→33

## 0.1.0 — 2026-07-10

- 初版コア: `Int/Float/Bool/String/List[T]`、中置演算子＋優先順位、`if` 式、`let..in`、再帰、契約（`requires/ensures/eg`）、構造化診断（スコープ開示）、JS バックエンド、CLI（`check/run/fmt/scope/emit-js`）
- 2バックエンドを同一の適合テストで一致検証する体制（Go の手順に倣う）
- 適合テスト 30 件
